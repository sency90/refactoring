# Restaurant Booking KATA

## 목적

- 식당 예약 도메인을 예제로 OOP/TDD 연습
- Mock을 활용한 설계 실습
- SOLID 원칙 의식하면서 코딩해보기
- 테스트 가능한 구조로 짜는 습관 만들기

## 시나리오 (점진적으로 확장 예정)

- 손님이 날짜/시간/인원으로 예약을 한다
- 좌석 수 초과 예약은 불가
- 동일 시간대 중복 예약 막기
- 예약 리스트 조회 (시간순)
- 관리자 입장에서 좌석 수 조정 가능

## 내가 집중할 포인트

- 도메인과 애플리케이션 로직 분리
- Repository 같은 추상화 어떻게 할지
- Mock 객체 어디까지 써야 적절한지
- 객체 간 의존성 줄이기
- 책임 분리 명확히 하기 (SRP, DIP 의식)

## 사용할 스택

- 언어: C++  
- 테스트: GoogleTest + gMock

## 진행 방식

- 작은 단위로 쪼개서 TDD
- 기능 하나씩 → 테스트 → 리팩토링 → 다음 단계
- 설계가 망했다 싶으면 중간에 구조 갈아엎기 허용
